## üö® CRITICAL ARCHITECTURAL CONSTRAINT üö®

**THIS PROJECT IS DESKTOP-FIRST. MOBILE IS A SUPPLEMENTAL VARIANT.**

### Non-Negotiable Rules:
1. **Desktop components MUST remain 100% intact and unchanged**
2. **Desktop performance MUST NOT decrease in any way**
3. **Mobile = NEW separate components, NOT modifications to desktop**
4. **All mobile work uses conditional rendering** (e.g., `isMobile ? <MobileComponent /> : <DesktopComponent />`)
5. **NO responsive Tailwind classes** (sm:, md:, lg:) on existing desktop components
6. **Desktop-first philosophy**: If conflict arises, desktop wins ALWAYS
7. **üî• CRITICAL: Content Parity Rule** - Mobile MUST use EXACT same content as desktop
   - ‚úÖ Use EXACT same translation keys as desktop
   - ‚úÖ Use EXACT same data sources (API, state, props)
   - ‚úÖ Keep EXACT same text, testimonials, numbers, statistics
   - ‚úÖ Adapt ONLY layout/presentation (grid ‚Üí carousel, 4-col ‚Üí 2-col)
   - ‚ùå NEVER create new content for mobile
   - ‚ùå NEVER make new testimonials/quotes/data
   - ‚ùå NEVER use different translation keys than desktop

**Implementation Pattern:**
```typescript
// ‚úÖ CORRECT - Create separate mobile components
{isMobile ? <SimplifiedHeroMobile /> : <Hero />}

// ‚ùå WRONG - Modify desktop components
<Hero className="hidden md:block" />  // This affects desktop!
```

**Testing Requirements:**
- Before every commit: Verify desktop works exactly as before
- Desktop performance must be equal or better
- Mobile components must be fully separate

See [MOBILE-ARCHITECTURE-PRINCIPLES.md](mdc:MOBILE-ARCHITECTURE-PRINCIPLES.md) for complete guidelines.

## üé® Mobile-Specific Design Patterns

### 1. **Desktop Superiority Messaging**
**Placement:** Top banner or cards
**Variants:**
- "üíª Best experience on desktop - Full interactive demo"
- "üñ•Ô∏è View on desktop for the complete experience"
- "üíª Desktop recommended for interactive features"

**Design:**
- Subtle, non-intrusive
- Blue/purple gradient background
- Dismissible (with cookie)
- Not a blocking modal
- Link to "Email me the desktop link"

### 2. **Sticky Bottom CTA Bar**
**Features:**
- Always visible while scrolling
- Two clear options: Primary + Secondary
- Large touch targets (min 48px)
- Doesn't obstruct content
- Hides on scroll down, shows on scroll up

**Example:**
```tsx
<StickyBottomBar
  primary={{ text: "Try Demo", onClick: handleDemo }}
  secondary={{ text: "Book Call", onClick: openCalendly }}
  autoHide={true}
/>
```

### 3. **Progressive Feature Reveal**
**Pattern:** Accordion or expandable cards
```tsx
<FeatureCard collapsed>
  <Icon + Title + OneLiner />
  <ExpandButton> // Shows full description
</FeatureCard>
```

### 4. **Touch-Optimized Interactions**
- All buttons ‚â• 48x48px (Apple HIG guideline)
- All nav items ‚â• 56px height with spacing
- Swipe gestures for carousels
- No hover states (use active/pressed states)
- Clear tap feedback (ripple effect or scale)

### 5. **Reduced Animation**
- Respect `prefers-reduced-motion`
- Simpler animations (fade/slide only)
- No complex 3D on mobile
- Faster animation durations (200-300ms vs 500-800ms)

## üîß Technical Implementation Strategy

### A. Responsive Breakpoints
```tsx
// Use consistent breakpoints across all components
const BREAKPOINTS = {
  mobile: '0px',        // 0-639px
  tablet: '640px',      // 640-1023px
  desktop: '1024px',    // 1024px+
}

// Tailwind classes
'hidden lg:block'  // Desktop only
'block lg:hidden'  // Mobile only
```

### B. Component Variants
**Pattern: Same component, different renders**
```tsx
export const Navigation = () => {
  const isMobile = useMediaQuery('(max-width: 1023px)')
  
  if (isMobile) {
    return <MobileBottomNav items={mobileNavItems} />
  }
  
  return <DesktopFloatingNav items={allNavItems} />
}
```

### C. Conditional Feature Loading
```tsx
// Don't load heavy components on mobile
{!isMobile && <CoreSphere3D />}
{isMobile && <StaticInfographic />}
```

### D. Touch Event Handling
```tsx
// Add touch event listeners where needed
<div
  onTouchStart={handleTouchStart}
  onTouchMove={handleTouchMove}
  onTouchEnd={handleTouchEnd}
>
```

### E. Performance Optimization
- Lazy load below-fold mobile content
- Smaller image assets for mobile
- Reduce bundle size for mobile entry points
- Service worker for offline mobile experience
