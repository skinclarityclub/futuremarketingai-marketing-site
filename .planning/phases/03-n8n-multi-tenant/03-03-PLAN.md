<task type="auto">
  <name>Task 2: Create Agency Client Setup V1.0 workflow</name>
  <files>n8n workflow: Agency Client Setup V1.0 (new)</files>
  <action>
Use mcp__n8n-docs__n8n_create_workflow to create a new workflow named "Agency Client Setup V1.0" with these nodes:

1. **"Webhook" trigger node**:
   - Method: POST
   - Path: `/agency-client-setup`
   - Accepts JSON body: `{ client_id, agency_id, trigger_first_run: boolean }`
   - This webhook URL will be called by the dashboard when an agency activates a new client workspace

2. **"Load Client Config" HTTP Request node**:
   - GET `{{$env.SUPABASE_URL}}/rest/v1/client_config?client_id=eq.{{$json.body.client_id}}`
   - Loads all config entries for this client

3. **"Load Client Accounts" HTTP Request node**:
   - GET `{{$env.SUPABASE_URL}}/rest/v1/client_accounts?client_id=eq.{{$json.body.client_id}}`
   - Loads social accounts for this client

4. **"Load Client Pillars" HTTP Request node**:
   - GET `{{$env.SUPABASE_URL}}/rest/v1/client_pillars?client_id=eq.{{$json.body.client_id}}`
   - Loads content pillars

5. **"Validate Config Completeness" Code node**:

   ```javascript
   const config = $('Load Client Config')
     .all()
     .map((i) => i.json)
   const accounts = $('Load Client Accounts')
     .all()
     .map((i) => i.json)
   const pillars = $('Load Client Pillars')
     .all()
     .map((i) => i.json)

   const required = ['brand_voice', 'target_audience', 'content_style']
   const configKeys = config.map((c) => c.config_key)
   const missingConfig = required.filter((r) => !configKeys.includes(r))

   const issues = []
   if (missingConfig.length > 0) issues.push(`Missing config: ${missingConfig.join(', ')}`)
   if (accounts.length === 0) issues.push('No social accounts configured')
   if (pillars.length === 0) issues.push('No content pillars defined')

   return [
     {
       json: {
         client_id: $input.first().json.body.client_id,
         is_valid: issues.length === 0,
         issues,
         config_count: config.length,
         accounts_count: accounts.length,
         pillars_count: pillars.length,
         trigger_first_run: $input.first().json.body.trigger_first_run,
       },
     },
   ]
   ```

6. **"Config Valid?" IF node**:
   - Condition: `{{$json.is_valid === true}}`
   - True branch: proceeds to optional first run trigger
   - False branch: returns validation errors

7. **"Should Trigger First Run?" IF node** (on true branch):
   - Condition: `{{$json.trigger_first_run === true}}`
   - True: triggers R&P Orchestrator
   - False: returns success without triggering

8. **"Trigger First R&P Run" HTTP Request node** (on trigger=true branch):
   - POST to R&P Orchestrator webhook URL with `{ client_id: $json.client_id }`
   - This triggers a single-client R&P pipeline run for the new client

9. **"Return Validation Result" Respond to Webhook node** (on false/invalid branch):
   - Returns: `{ success: false, issues: $json.issues }`

10. **"Return Setup Success" Respond to Webhook node** (on valid branches):
    - Returns: `{ success: true, client_id: $json.client_id, first_run_triggered: $json.trigger_first_run }`

Set workflow to ACTIVE.

Note: The webhook URL generated by n8n will need to be configured in the dashboard's client onboarding flow (Phase 2 DASH-04/DASH-07). Record the webhook URL in the summary for cross-phase reference.
</action>
