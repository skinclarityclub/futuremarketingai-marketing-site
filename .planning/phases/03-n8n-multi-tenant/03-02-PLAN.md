  <name>Task 1: Parameterize Daily Analytics Collector for multi-client</name>
  <files>n8n workflow: Daily Analytics Collector V1.0 (tIwqBmpgzNGZnVKT)</files>
  <action>
First, read the current workflow using mcp__n8n-docs__n8n_get_workflow with workflow ID `tIwqBmpgzNGZnVKT` to understand the 13-node structure and identify hardcoded IG queries.

Then modify the workflow to add multi-client support:

1. **Add "Fetch Active Clients" HTTP Request node** after the Schedule Trigger:
   - Method: GET
   - URL: `{{$env.SUPABASE_URL}}/rest/v1/clients?is_active=eq.true&select=id,name`
   - Headers: apikey, Authorization (service_role pattern)

2. **Add "Loop Clients" SplitInBatches node** after Fetch Active Clients:
   - Batch Size: 1 (process one client at a time to avoid API rate limits)

3. **Add "Load Client IG Token" HTTP Request node** inside the loop:
   - URL: `{{$env.SUPABASE_URL}}/rest/v1/client_config?client_id=eq.{{$json.id}}&config_key=eq.instagram_access_token`
   - This replaces any hardcoded IG access token

4. **Add "Skip If No Token" IF node** after Load Client IG Token:
   - Condition: `{{$json.length > 0}}` (true branch continues, false branch skips to loop-back)
   - This gracefully handles clients without IG tokens configured (research recommendation)

5. **Modify existing IG API query nodes** to use dynamic token:
   - Replace hardcoded access_token with: `{{$('Load Client IG Token').first().json.config_value}}`
   - Handle double JSON.parse: if config_value is string, parse it first

6. **Scope all Supabase data queries to client_id**:
   - Any queries to content_items, content_briefs, or analytics tables must include `client_id=eq.{{$json.client_id}}` filter

7. **Add "Load Notification Config" for error handling** (per N8N-08):
   - Load telegram_bot_token and telegram_chat_id from client_config for this client
   - Update any existing error notification nodes to use dynamic config with SKC fallback

8. **Wire the loop-back**: Connect the end of the analytics processing chain back to the "Loop Clients" SplitInBatches node.

Keep workflow INACTIVE (do not activate). Use mcp**n8n-docs**n8n_update_partial_workflow for modifications.
