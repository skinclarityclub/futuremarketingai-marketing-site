<objective>
Eliminate eager-load of ChatWidget, CalendlyModal, BookingModal, CookieConsentBanner, and Spline prefetch on every page. Replace with click-driven lazy imports plus home-only Spline hints. Trim runtime cost of gradient blobs and HeaderClient listeners.

Purpose: Today, 95 percent of visitors never open the chat or book a call, yet every page ships roughly 71 KB gzipped of chat + Calendly + booking code, 1.3 MB of Spline prefetch on 86 non-home pages, and three 600 px blurred blobs running infinite animations on every route. This plan strips all that waste.

Output: Non-home pages drop roughly 50 KB gz of initial JS (chat + Calendly + booking + cookie), 1.3 MB of bandwidth (Spline), and sustained GPU composition work (blobs). Home page stays visually identical; hero Spline still loads as before.
</objective>

**Part B**: Inside `CookieConsentBanner.tsx`, short-circuit the render if the cookie `futuremarketingai-cookie-consent` is already set. This stops react-cookie-consent from evaluating and attaching listeners for returning visitors who already accepted.

Add near the top of the component body (after `useState(false) mounted` and the `useEffect` that sets it):

```tsx
const [hasConsent, setHasConsent] = useState<boolean | null>(null)

useEffect(() => {
  setMounted(true)
  const cookies = typeof document !== 'undefined' ? document.cookie : ''
  setHasConsent(cookies.includes('futuremarketingai-cookie-consent='))
}, [])

if (!mounted) return null
if (hasConsent) return null
// ...existing render...
```

Result: on mount, if cookie is present, the entire `CookieConsent` library subtree is never rendered, so react-cookie-consent's module cost is still paid once (dynamic import happens) but the runtime init is skipped. On dynamic-import-miss path (no cookie), the banner renders as before.

For returning visitors with cookie already set, the chunk still downloads once when `ClientIslands` mounts the dynamic component. To save that too, extend `ClientIslands` to check the cookie BEFORE calling the dynamic import. Safer alternative: render `<CookieConsentBannerLazy />` only conditionally based on a synchronous `document.cookie` check wrapped in a `useState` + `useEffect`:

```tsx
// In ClientIslands.tsx
const [needsConsent, setNeedsConsent] = useState(false)
useEffect(() => {
  if (typeof document !== 'undefined' && !document.cookie.includes('futuremarketingai-cookie-consent=')) {
    setNeedsConsent(true)
  }
}, [])
// then conditionally render:
{needsConsent && <CookieConsentBannerLazy />}
```

This second approach means the react-cookie-consent chunk is never even requested for returners. Prefer this; it is the full 8 KB gz savings.

// Close dropdown on Escape or outside click — only while open
useEffect(() => {
  if (!skillsOpen) return

    if (e.key === 'Escape') setSkillsOpen(false)
