<task type="auto">
  <name>Task 2: Extend prefers-reduced-motion coverage to all keyframes and inline animations</name>
  <files>fmai-nextjs/src/app/globals.css</files>
  <action>
Replace the existing narrow block at `fmai-nextjs/src/app/globals.css:308-314`:

```css
@media (prefers-reduced-motion: reduce) {
  .blob-warm,
  .blob-cool,
  .blob-mixed {
    animation: none;
  }
}
```

with a comprehensive block that disables all 17 registered keyframes plus any inline style-attribute animation:

```css
@media (prefers-reduced-motion: reduce) {
  /*
   * WCAG 2.3.3 Animation from Interactions (AAA) + 2.3.1 Three Flashes (A).
   * Universal selector with !important overrides inline `style="animation: fadeInUp ..."`
   * on src/app/[locale]/page.tsx:87,96,110,118,126 and any other inline animation.
   * The 0.01ms duration preserves `animationend` events for JS listeners while
   * visually halting motion.
   */
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    animation-delay: 0ms !important;
    transition-duration: 0.01ms !important;
    transition-delay: 0ms !important;
    scroll-behavior: auto !important;
  }

  /* Explicit halt for CSS-class-based animations that should stop entirely */
  .blob-warm,
  .blob-cool,
  .blob-mixed,
  .loader {
    animation: none !important;
  }
}
```

The `0.01ms` duration trick (universal pattern documented by the W3C and Andy Bell) is used because:

- Inline styles like `style={{ animation: 'fadeInUp 0.8s ease-out 0.2s both' }}` cannot be overridden without `!important` in a stylesheet rule.
- Animations with fill-mode `both` land at their terminal state, so setting `animation-duration: 0.01ms` causes them to snap into place instantly without disappearing (no content-blanking).
- The `.blob-*` and `.loader` classes use decorative animation; they are set to `none` so they do not "snap" in either — they simply freeze at their default frame.

Note: Framer Motion respects `prefers-reduced-motion` separately via the `useReducedMotion()` hook (used at e.g. `FloatingButton.tsx:17`). No change needed in JS components.
