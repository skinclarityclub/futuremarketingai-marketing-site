**Create `src/components/chatbot/demo/DemoOrchestrator.tsx`:**

The core state machine component that orchestrates the guided demo flow. Rendered inside ChatWidget.

Props: `chatStatus` (string from useChat), `onSendMessage` (callback to send messages via useChat)

Key behaviors:

**1. Sending scripted messages:**

- When `demoStatus === 'running'` AND `chatStatus === 'ready'` AND message not yet sent → send `currentStep.userMessage` via `onSendMessage` after 500ms delay
- Use `hasSentRef` ref to prevent double-sends. Reset it when `demoStepIndex` changes.

**2. Detecting AI completion:**

- After sending a message, wait for `chatStatus` to return to `'ready'`
- Then check: is this the last step? → set status to 'completed'
- Does this step have a checkpoint? → set status to 'checkpoint'
- Otherwise → auto-advance (reset hasSentRef, call advanceStep)
- Use 300ms delay after AI ready before checking (let content settle)

**3. Checkpoint handling:**

- `'next'` → call advanceStep
- `'skip-to-booking'` → send BOOKING_STEP.userMessage, set status to 'completed'
- `'next-scenario'` → set status to 'choosing'
- `'end'` → call endDemo

**4. Rendering:**

- When `demoStatus === 'choosing'` → render DemoScenarioCard for each scenario
- When `demoStatus === 'checkpoint'` → render DemoCheckpoint with current step's checkpoint config
- When `demoStatus === 'completed'` → render DemoCompletionCard with scenario title, duration, and callbacks
- When `demoMode === false` → return null

Use the exact implementation from `docs/plans/2026-03-16-guided-demo-implementation.md` Task 8.
