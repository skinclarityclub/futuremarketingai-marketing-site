## Bugs Fixed (from user testing)

- **Chat panel under header**: z-index `z-50` → `z-[60]`, positioned `top-[80px]` (below 64px header)
- **FloatingButton hidden behind panel**: z-index `z-[51]` → `z-[61]`
- **Side panel wrong direction**: Moved from RIGHT to LEFT of chat in JSX order, `border-l` → `border-r`, animation `x:20` → `x:-20`
- **Side panel too narrow**: 440px → `min(520px, 40vw)` with `min-w-[280px]` (responsive)
- **Content overflowing viewport**: Added `max-w-[calc(100vw-3rem)]` and `max-h-[calc(100vh-104px)]` + `lg:bottom-6` on dialog container
- **Chat panel height**: Changed from fixed `h-[80vh] max-h-[720px]` to `h-full` (fills container)

## Side Panel Dynamic Updates (critical fix)

- **Race condition in SidePanelTrigger**: Removed all `useEffect` side effects from individual `SidePanelTrigger` components. Now a pure render component (just "View details" button).
- **Centralized auto-open**: Single `useMemo` + `useEffect` in `ChatMessages` finds the LAST tool result from the last assistant message and calls `openSidePanel`. Eliminates race conditions.
- **Extracted `getToolName` helper**: Shared between useMemo and render loop for DRY tool name extraction.
- **All tools now route to side panel**: Moved `get_services`, `create_ticket`, `check_status`, `escalate_to_human` from `INLINE_TOOLS` to `SIDE_PANEL_TOOLS`. Only `navigate_to_page` remains inline.

## Data Format Mismatches Fixed (5 tools)

1. **qualify_lead**: `qualification` → `stage`, `nextSteps` → `factors` (fallback logic)
2. **get_pricing_info**: Object tiers `{ starter: { monthlyPrice: 299 } }` → array `[{ price: "€299/mo" }]`
3. **get_roi_estimate**: `roiPercent` → `roi` ("463%"), `paybackPeriodMonths` → `paybackPeriod` ("1 month"), `breakdown.hoursSavedPerWeek` → `hoursReclaimed`
4. **escalate_to_human**: `action: 'escalate'` → `escalated: true`, `estimatedWait` → `estimatedResponse`
5. **get_roi_info**: Raw fields auto-construct `metrics[]` array
6. **get_services**: Object format → array, `highlights` → `features`, `starting_price` → `price`
7. **explain_module**: Nested `module: { name, ... }` object → extracted fields

## Tool Result Cards Upgraded

- **ServiceCard**: Supports multi-service listings, pricing tiers with "Popular" badge, module explanation with metrics grid, icon mapping per service type
- **LeadScoreCard**: Animated score, ROI projection with metrics grid, assessment factors, handles both score and ROI-only formats
- **TicketCard**: Status icons, priority badges, escalation view with agent info, timeline
- **KBArticleCard**: Relevance progress bar, category badges, multi-article listings

## AI Response Length

- **maxTokens**: 600 → 250
- **System prompt**: "1-2 sentences max, let the tool cards do the talking" with good/bad examples

## Demo Pacing (user-driven)

- **Removed all auto-advance timing**: No more `getTypingDelay()` or `getReadingDelay()` functions
- **New `awaiting-continue` status**: Added to Zustand store's `demoStatus` union type
- **Continue button**: After each AI response, shows a "Continue demo" button with step indicator. User clicks to advance.
- **Only typing delay remains**: Short 0.8-1.2s delay before scripted messages appear (simulates typing)

- **Playwright**: Removed Mobile Chrome and Mobile Safari projects (desktop-first per CLAUDE.md)
