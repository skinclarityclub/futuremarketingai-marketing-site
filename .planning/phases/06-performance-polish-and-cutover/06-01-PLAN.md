<objective>
Optimize the Next.js production bundle for Core Web Vitals green scores by lazy-loading the heaviest client islands (chatbot, Calendly) and configuring bundle analysis tooling.

Purpose: The chatbot dependency tree (~20 "use client" files + AI SDK + streaming + Zustand store) is statically imported in the locale layout, bloating every page's initial load. Converting to dynamic imports removes it from the critical rendering path, directly improving LCP and INP.

Output: Optimized layout.tsx with dynamic imports, bundle analyzer configured in next.config.ts, production build with reduced First Load JS.
</objective>

<task type="auto">
  <name>Task 2: Dynamic import heavy client islands and verify CWV improvements</name>
  <files>fmai-nextjs/src/app/[locale]/layout.tsx</files>
  <action>
  1. In `fmai-nextjs/src/app/[locale]/layout.tsx`, replace the static imports for heavy client islands with `next/dynamic`:
     - Remove: `import { ChatWidgetIsland } from '@/components/chatbot/ChatWidgetIsland'`
     - Remove: `import { CalendlyIsland } from '@/components/interactive/CalendlyIsland'`
     - Add: `import dynamic from 'next/dynamic'`
     - Add: `const ChatWidgetIsland = dynamic(() => import('@/components/chatbot/ChatWidgetIsland').then(mod => mod.ChatWidgetIsland), { ssr: false })`
     - Add: `const CalendlyIsland = dynamic(() => import('@/components/interactive/CalendlyIsland').then(mod => mod.CalendlyIsland), { ssr: false })`
     Note: Both components use named exports, so `.then(mod => mod.ComponentName)` is required.

2. Verify CookieConsentBanner uses `position: fixed` or `position: sticky` at the bottom of the viewport (not in normal document flow). If it pushes content, wrap it similarly with dynamic import or ensure its CSS prevents CLS. Read the component to check.

3. Check if any hero images on the homepage use next/image -- if so, verify the `priority` prop is set on above-fold images. The project likely uses text-only heroes (h1), in which case font loading (already optimized with next/font display:swap) handles LCP.

4. Run `npm run build` and compare First Load JS sizes against the baseline from Task 1. The shared chunk size should decrease since chatbot and calendly bundles are now lazy-loaded.
