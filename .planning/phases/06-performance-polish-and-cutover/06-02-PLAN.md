<task type="auto">
  <name>Task 1: Structured data audit, sitemap/hreflang verification, and Lighthouse CWV audit</name>
  <files>fmai-nextjs/src/app/sitemap.ts, fmai-nextjs/src/app/robots.ts</files>
  <action>
  1. Build and start the production server:
     ```
     cd fmai-nextjs && npm run build && npm run start
     ```

2. Verify sitemap completeness:
   - Fetch `http://localhost:3000/sitemap.xml`
   - Count total URLs -- expected: 14 pages x 3 locales = 42 entries minimum (plus blog posts)
   - Verify each page entry has all 3 locale alternates (en, nl, es) including self-referencing hreflang
   - Check that blog post URLs are included
   - If any pages or alternates are missing, fix sitemap.ts

3. Verify robots.txt:
   - Fetch `http://localhost:3000/robots.txt`
   - Confirm it includes: Sitemap URL, AI crawler allow-policy (allow retrieval bots like GPTBot, block training bots)
   - If the sitemap URL points to localhost or incorrect domain, fix robots.ts to use the production domain

4. Audit all JSON-LD structured data by fetching page source for key pages and validating the script[type="application/ld+json"] tags:
   - Homepage (`/en`): Organization + WebSite (with SearchAction) + WebPage + BreadcrumbList
   - Service page (`/en/automations`): Organization + Service + WebPage + BreadcrumbList + FAQPage
   - How It Works (`/en/how-it-works`): Organization + WebPage + BreadcrumbList + HowTo
   - Blog listing (`/en/blog`): Organization + WebPage + BreadcrumbList
   - Blog post (`/en/blog/[first-slug]`): Organization + Article (with datePublished, dateModified, author Person) + BreadcrumbList
   - About (`/en/about`): Organization + WebPage + BreadcrumbList
   - Pricing (`/en/pricing`): Organization + WebPage + BreadcrumbList

5. For each page, validate JSON-LD parses correctly (valid JSON) and contains required fields per schema.org spec. Fix any issues found:
   - Organization: name, url, logo
   - WebSite: name, url, potentialAction (SearchAction)
   - Service: name, description, provider
   - FAQPage: mainEntity array with Question/acceptedAnswer
   - HowTo: name, step array
   - Article: headline, datePublished, dateModified, author
   - BreadcrumbList: itemListElement array

6. Run Lighthouse CLI on 3 key pages to measure CWV:

   ```
   npx lighthouse http://localhost:3000/en --output json --quiet --chrome-flags="--headless" 2>/dev/null
   npx lighthouse http://localhost:3000/en/chatbots --output json --quiet --chrome-flags="--headless" 2>/dev/null
   npx lighthouse http://localhost:3000/en/blog --output json --quiet --chrome-flags="--headless" 2>/dev/null
   ```

   Extract: Performance score, LCP, INP (TBT as proxy), CLS for each page.
   Target: LCP <2.5s, CLS <0.1, Performance score 90+.

7. If any CWV metric is out of range, diagnose and fix:
   - LCP >2.5s: Check for large above-fold images missing `priority`, or render-blocking resources
   - CLS >0.1: Check ScrollReveal components, CookieConsentBanner positioning, font loading
   - High TBT: Check for heavy synchronous JavaScript execution (likely chatbot if not properly lazy-loaded)
