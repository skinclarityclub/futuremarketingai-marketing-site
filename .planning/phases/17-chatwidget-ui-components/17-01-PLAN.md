# Plan 01: Foundation — Engine Compatibility, Chat Store, usePersonaChat Hook

## Objective

Set up the three foundation layers the ChatWidget UI depends on:

1. Switch engine from `toTextStreamResponse` to `toUIMessageStreamResponse` so `useChat` receives structured message parts (text + tool invocations)
2. Create the new `chatbotStore.ts` (Zustand) for per-persona UI state (separate from old ARIA `chatStore.ts`)
3. Create `usePersonaChat` hook wrapping AI SDK v6 `useChat` with `DefaultChatTransport`

**Create `src/stores/chatbotStore.ts`** — New Zustand store for chatbot UI state. This is a SEPARATE file from the old `chatStore.ts` (ARIA) which is removed in Phase 19. Use export name `useChatbotStore` to avoid conflicts.

Store shape:

```ts
interface ChatbotState {
  // Active persona
  activePersonaId: string

  // UI state (floating mode)
  isOpen: boolean
  isMinimized: boolean
  hasUnread: boolean

  // Session
  sessionId: string
  messageCount: number

  // Actions
  setActivePersona: (id: string) => void
  open: () => void
  close: () => void
  minimize: () => void
  toggle: () => void
  markRead: () => void
  setUnread: () => void
  incrementMessageCount: () => void
  resetSession: () => void
}
```

Implementation details:

- Use `create` from `zustand` and `persist` from `zustand/middleware`
- localStorage key: `fmai-chatbot-state` (different from old `fmai-chat-state`)
- Generate initial sessionId via `crypto.randomUUID()` (fallback: `Date.now().toString(36) + Math.random().toString(36).slice(2)`)
- Partialize: only persist `sessionId`, `messageCount`, `activePersonaId` — NOT UI state (isOpen, isMinimized)
- Default activePersonaId: `'concierge'`
- `resetSession` generates new sessionId and resets messageCount to 0
- `incrementMessageCount` adds 1 to messageCount

**Create `src/hooks/usePersonaChat.ts`** — Wrapper around AI SDK v6 `useChat` hook.

```ts
import { useChat } from '@ai-sdk/react'
import { DefaultChatTransport } from 'ai'
import { useChatbotStore } from '../stores/chatbotStore'

export function usePersonaChat(personaId: string) {
  const { sessionId, messageCount, incrementMessageCount } = useChatbotStore()

  const chat = useChat({
    id: `chat-${personaId}`,
    transport: new DefaultChatTransport({
      api: '/api/chatbot',
      body: {
        personaId,
        sessionId,
      },
    }),
    onFinish: () => {
      incrementMessageCount()
    },
    onError: (error) => {
      console.error(`[chat-${personaId}]`, error)
    },
  })

  return {
    ...chat,
    messageCount,
    isAtLimit: messageCount >= 15,
  }
}
```

Key details:

- `useChat` from `@ai-sdk/react` (NOT from `ai` — the React hook is in the React package)
- `DefaultChatTransport` from `ai` package
- The `id` parameter (`chat-${personaId}`) creates per-persona message isolation — useChat manages message state internally
- `body` includes `personaId` and `sessionId` which the engine reads alongside the `messages` array
- `onFinish` increments the Zustand message counter (for demo limits)
- Return spread of useChat plus `messageCount` and `isAtLimit` for UI consumption
- `status` from useChat replaces the old `isLoading` — check `status !== 'ready'` for loading state
- `sendMessage({ text })` is how to send — NOT handleSubmit (removed in v6)
