1. Create `src/lib/chatbot/topic-router.ts`:

   Import `TopicDefinition`, `TopicRouterResult` from `./types`.

   Helper `escapeRegex(str: string): string` — escape special regex characters.

   `routeToKnowledge(message: string, topics: TopicDefinition[], conversationTopics?: string[]): TopicRouterResult`:
   - Lowercase the message
   - For each topic, score by keyword matching:
     - +1 for each keyword found via `lower.includes(keyword.toLowerCase())`
     - +0.5 bonus for exact word boundary match (RegExp `\b...\b`)
     - +0.3 for context carryover (topic.key in conversationTopics)
   - Filter to topics with score >= 1
   - Sort by score DESC, then priority DESC as tiebreaker
   - Select top 3
   - Return `{ topics: selected keys, knowledgeContent: selected contents joined with \n\n, estimatedTokens: ceil(total content length / 4) }`
   - If no topics match, return empty topics array, empty knowledgeContent, 0 estimatedTokens

   This is the SKC sindy-topic-router pattern adapted for generic persona use.

2. Create `src/lib/chatbot/prompt-builder.ts`:

   Import `PersonaConfig`, `TopicRouterResult` from `./types`.

   Define `SystemMessage` interface locally:
   ```typescript
   interface SystemMessage {
     role: 'system'
     content: string
     providerOptions?: {
       anthropic?: { cacheControl?: { type: 'ephemeral' } }
     }
   }
````

`buildSystemMessages(persona: PersonaConfig, topicResult: TopicRouterResult, context?: { language?: string; currentPage?: string }): SystemMessage[]`:

- Part 1: Static prefix with cache control (ALWAYS included):
  ```
  { role: 'system', content: persona.staticPrefix, providerOptions: { anthropic: { cacheControl: { type: 'ephemeral' } } } }
  ```
- Part 2: Dynamic knowledge (only if topicResult.knowledgeContent is non-empty):
  ```
  { role: 'system', content: `## Relevant Knowledge:\n${topicResult.knowledgeContent}` }
  ```
  NO cacheControl on this — it changes per request.
- Part 3: Request context (only if context provided):
  ```
  { role: 'system', content: `## Context:\n- Language: ${context.language ?? 'en'}\n- Current page: ${context.currentPage ?? 'unknown'}` }
  ```
  NO cacheControl.

Per RESEARCH.md Pitfall 2: static prefix must be BYTE-FOR-BYTE identical across requests for same persona. No timestamps, no dynamic content in Part 1.

Per RESEARCH.md Pitfall 3: Haiku requires 4096 tokens minimum for caching. The cache control header is still set (harmless) — Anthropic silently ignores it below threshold. This is acceptable since Haiku is already cheap.
</action>

1. Create `src/lib/chatbot/persona-router.ts`:

Import `PersonaConfig` from `./types`.

Module-level `PERSONA_REGISTRY = new Map<string, PersonaConfig>()`.

`registerPersona(config: PersonaConfig): void`:

- Validate config.id is non-empty string
- Validate config.staticPrefix is non-empty string
- Set in registry: `PERSONA_REGISTRY.set(config.id, config)`

`getPersona(personaId: string): PersonaConfig`:

- Get from registry
- If not found, throw `new Error(\`Persona not found: ${personaId}. Available: ${[...PERSONA_REGISTRY.keys()].join(', ')}\`)`
- Return config

`hasPersona(personaId: string): boolean`:

- Return `PERSONA_REGISTRY.has(personaId)`

`getRegisteredPersonaIds(): string[]`:

- Return `[...PERSONA_REGISTRY.keys()]`

NOTE: Personas are registered by Phase 16. This is just the registry infrastructure. The registry is empty until persona modules call `registerPersona()`.

2. Create `src/lib/chatbot/tool-executor.ts`:

   Import `PersonaConfig` from `./types`.
   Import `tool } from 'ai'`.
   Import `z` from `zod`.

   `createPersonaTools(persona: PersonaConfig): Record<string, ReturnType<typeof tool>>`:
   - This is a factory that takes a PersonaConfig and returns the AI SDK `tool()` wrapped versions of the persona's tools
   - For Phase 15, return an empty record `{}` — actual tool definitions come in Phase 16
   - This establishes the pattern: persona.tools contains raw tool configs, this function wraps them with `tool()` from AI SDK

   `executeToolCall(personaId: string, toolName: string, args: unknown): Promise<unknown>`:
   - Import `getPersona` from `./persona-router`
   - Get persona config
   - Check if `toolName` exists in `persona.tools`
   - If not, throw `new Error(\`Tool ${toolName} not available for persona ${personaId}\`)`
   - This is the manual execution path (backup if streamText auto-execution is not used)
   - For Phase 15, return `{ error: 'Tool execution not yet implemented' }` — Phase 16 fills in actual handlers

   Export a `DEMO_TOOL` as a reference implementation showing the AI SDK `tool()` pattern:

     }),
     execute: async ({ input }) => ({ echo: input, timestamp: new Date().toISOString() }),
   })

   This serves as documentation for Phase 16 implementers showing the exact `tool()` + Zod pattern.
