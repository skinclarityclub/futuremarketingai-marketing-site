Create `src/lib/chatbot/engine.ts`:

CORS headers constant:

```typescript
const CORS_HEADERS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Methods': 'POST, OPTIONS',
  'Access-Control-Allow-Headers': 'Content-Type',
} as const
```

Helper `getClientIp(request: Request): string`:

- Check `x-forwarded-for` header, take first IP
- Check `x-real-ip` header
- Fallback to `'unknown'`

`handleChatRequest(request: Request): Promise<Response>`:

1. Handle OPTIONS preflight:

   ```typescript
   if (request.method === 'OPTIONS') {
     return new Response(null, { status: 200, headers: CORS_HEADERS })
   }
   ```

2. Reject non-POST:

   ```typescript
   if (request.method !== 'POST') {
     return Response.json({ error: 'Method not allowed' }, { status: 405, headers: CORS_HEADERS })
   }
   ```

3. Parse request body. Wrap in try/catch for JSON parse errors:

   ```typescript
   let body: ChatRequest
   try {
     body = (await request.json()) as ChatRequest
   } catch {
     return Response.json({ error: 'Invalid JSON body' }, { status: 400, headers: CORS_HEADERS })
   }
   ```

4. Destructure: `const { personaId, message, sessionId, conversationHistory, context } = body`

5. Validate required fields (personaId, message, sessionId must be non-empty strings). Return 400 if missing.

6. Validate input via security module:

   ```typescript
   const validation = validateInput(message)
   if (!validation.valid) {
     return Response.json({ error: validation.reason }, { status: 400, headers: CORS_HEADERS })
   }
   ```

7. Check rate limits:

   ```typescript
   const ip = getClientIp(request)
   const rateCheck = checkAllRateLimits(sessionId, ip)
   if (!rateCheck.allowed) {
     return Response.json(
       { error: 'Rate limit exceeded', limitType: rateCheck.limitType, resetAt: rateCheck.resetAt },
       { status: 429, headers: CORS_HEADERS }
     )
   }
   ```

8. Load persona (wrap in try/catch, return 404 if not found):

   ```typescript
   let persona
   try {
     persona = getPersona(personaId)
   } catch {
     return Response.json(
       { error: `Unknown persona: ${personaId}` },
       { status: 404, headers: CORS_HEADERS }
     )
   }
   ```

9. Route knowledge via topic router:

   ```typescript
   const topicResult = routeToKnowledge(message, persona.topicDefinitions)
   ```

10. Build system messages with prompt caching:

    ```typescript
    const systemMessages = buildSystemMessages(persona, topicResult, context)
    ```

11. Detect complexity for model routing:

    ```typescript
    const complexity = detectComplexity(
      message,
      conversationHistory?.length ?? 0,
      persona.complexityKeywords
    )
    const modelId = MODEL_IDS[complexity]
    ```

12. Create persona tools:

    ```typescript
    const tools = createPersonaTools(persona)
    ```

13. Stream response:

    ```typescript
    const result = streamText({
      model: anthropic(modelId),
      messages: [
        ...systemMessages,
        ...(conversationHistory ?? []).map((msg) => ({
          role: msg.role as 'user' | 'assistant',
          content: msg.content,
        })),
        { role: 'user' as const, content: validation.sanitizedInput ?? message },
      ],
      tools: Object.keys(tools).length > 0 ? tools : undefined,
      toolChoice: Object.keys(tools).length > 0 ? 'auto' : undefined,
      maxTokens: persona.maxTokens,
      temperature: persona.temperature,
    })

    return result.toTextStreamResponse({
      headers: CORS_HEADERS,
    })
    ```

14. Wrap entire function body in outer try/catch for unexpected errors:
    ```typescript
    catch (error) {
      console.error('[chatbot-engine] Unexpected error:', error)
      return Response.json(
        { error: 'Internal server error' },
        { status: 500, headers: CORS_HEADERS }
      )
    }
    ```

Per RESEARCH.md: Use `toTextStreamResponse()` for simplicity (Open Question 1). Phase 17 can switch to `toUIMessageStreamResponse()` if `useChat` hook is adopted â€” it's a one-line change.
