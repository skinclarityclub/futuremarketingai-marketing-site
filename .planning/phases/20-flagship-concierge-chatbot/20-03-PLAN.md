**2. Create `src/hooks/useConciergeContext.ts`:**
This hook provides all context-aware behavior for the flagship concierge. It is used by App.tsx to derive ChatWidget props.

- Accept: `pathname: string`, `lang: string` (language code).
- Return: `{ welcomeMessage, suggestedPrompts, pageContext }`.

**PAGE_GREETINGS** (Record<string, Record<string, string>> -- path -> lang -> greeting):

```
'/': { en: 'Welcome to FMai! ...', nl: 'Welkom bij FMai! ...', es: 'Bienvenido a FMai! ...' }
'/chatbots': { en: 'Looking at our chatbot solutions? ...', ... }
'/pricing': { en: 'Exploring pricing? ...', ... }
'/marketing-machine': { en: 'Want to learn about the Marketing Machine? ...', ... }
'/automations': { en: 'Checking out our automation services? ...', ... }
'/voice-agents': { en: 'Interested in voice agents? ...', ... }
'/about': { en: 'Want to know more about FMai? ...', ... }
'/contact': { en: 'Ready to get in touch? ...', ... }
```

Default fallback for unknown pages.

**PAGE_PROMPTS** (Record<string, Record<string, string[]>> -- path -> lang -> prompts):

- Each page gets 3-4 contextual suggested prompts relevant to that page content.
- Example for '/pricing': ['Compare plans for me', 'What ROI can I expect?', 'Help me choose a plan']
- Example for '/chatbots': ['Show me an e-commerce chatbot demo', 'How do chatbots help with lead gen?', 'What platforms do you support?']
- Provide EN/NL/ES versions.

**Logic:**

- Use `useChatbotStore` to read `visitedPages` and `messageCounts`.
- On mount / pathname change: call `addVisitedPage(pathname)`.
- Derive `welcomeMessage` from PAGE_GREETINGS based on pathname + lang. If visitedPages.length > 2, append a personalized touch: "I see you've been exploring -- ask me anything!"
- Derive `suggestedPrompts` from PAGE_PROMPTS based on pathname + lang. If messageCounts['flagship'] > 3, switch to follow-up prompts (different set: 'Book a discovery call', 'Show me case studies', 'Compare your services').
- Return `pageContext: { pathname }` for the API.

**3. Add context-aware tool filtering in `src/lib/chatbot/engine.ts`:**

- Add a private function `filterToolsByContext(allTools, context)`:
  - `ALWAYS_AVAILABLE` tools: get_services, book_call, navigate_to_page, get_case_study
  - `PAGE_TOOLS` record mapping page paths to additional tool names (see RESEARCH.md Pattern 2 for the exact mapping):
    - '/chatbots': search_products, get_product_details, build_routine, search_knowledge_base, create_ticket
    - '/marketing-machine': explain_module, get_roi_info, get_roi_estimate
    - '/pricing': get_pricing_info, get_roi_estimate, qualify_lead
    - '/voice-agents': get_pricing_info, qualify_lead
    - '/automations': get_pricing_info, qualify_lead
    - '/contact': qualify_lead, create_ticket
  - If no context or no currentPage: return all tools (fallback).
  - Use Object.entries + forEach to filter (avoid for-of per project convention).
- Apply `filterToolsByContext` ONLY for the flagship persona. In the engine's step 12, after `createPersonaTools(persona)`:
  - If `persona.id === 'flagship'` and `context?.currentPage`: filter tools.
  - Otherwise: use all tools as-is.
- This reduces prompt size from ~16 tools to ~6-8 per request for flagship.
