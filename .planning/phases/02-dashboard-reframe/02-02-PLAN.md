**2. Create `src/lib/actions/billing.ts`.**

Create a new server action file with 'use server' directive.

`createCheckoutSession(tierKey: string)`:

- Validate tierKey against AGENT_TIERS keys using zod
- Get orgContext via getOrgContext()
- Fetch org's stripe_customer_id from fma_organizations via admin client
- If no customer_id: create Stripe customer with metadata.organization_id, save to org
- Create Stripe Checkout Session in subscription mode with:
  - line_items: [{ price: tier.priceId, quantity: 1 }]
  - metadata: { organization_id, tier: tierKey }
  - success_url: `${NEXT_PUBLIC_APP_URL}/billing?success=true`
  - cancel_url: `${NEXT_PUBLIC_APP_URL}/billing?canceled=true`
- Return { url: session.url }

`createPortalSession()`:

- Get orgContext, fetch stripe_customer_id
- Create Stripe Billing Portal session
- Return { url: session.url }

Use `createAdminClient` from `@/lib/supabase/server` for database operations (bypasses RLS).

**3. Create `src/app/api/webhooks/stripe/route.ts`.**

Create App Router route handler for Stripe webhooks.

POST handler:

- Read raw body via `req.text()`
- Get signature from `stripe-signature` header via `await headers()`
- Construct event via `stripe.webhooks.constructEvent(body, sig, STRIPE_WEBHOOK_SECRET)`
- Use admin Supabase client (no auth needed for webhooks)

Handle events:

- `checkout.session.completed`: Update org's stripe_customer_id, stripe_subscription_id, subscription_tier from session.metadata
- `customer.subscription.updated`: Sync tier changes. Read subscription items to determine current tier. Update org's subscription_tier.
- `customer.subscription.deleted`: Set org's subscription_tier to 'free', clear stripe_subscription_id
- `invoice.payment_failed`: Log warning (future: send notification to agency owner)

Use idempotency: check if stripe_subscription_id already matches before processing checkout.session.completed.

Return 200 for all events (even unhandled ones).

Ensure the directory `src/app/api/webhooks/stripe/` is created.
