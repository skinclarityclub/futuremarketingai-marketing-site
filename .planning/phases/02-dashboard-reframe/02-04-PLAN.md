**1. Create `src/lib/actions/onboarding.ts`** — server actions for each onboarding step.

'use server' directive. Three actions:

`saveAgencyInfo(input)`:

- Validate: { displayName: string min 2, website?: string url, industry?: string }
- Get orgContext, update fma_organizations with display_name, agency_website, agency_industry
- Return { success: true }

`createFirstClient(input)`:

- Validate: { name: string min 2 }
- Get orgContext
- Insert into fma_clients: { name, organization_id: orgCtx.organizationId, active_skills: '{}' }
- Return { success: true, clientId: newClient.id }

`completeOnboarding()`:

- Get orgContext
- Update fma_organizations: set onboarding_completed_at to NOW()
- revalidatePath('/dashboard')
- Return { success: true }

**2. Create `src/components/onboarding/agency-onboarding-wizard.tsx`** — client component.

'use client' directive. Multi-step wizard with 4 steps:

**Step 1: Agency Info**

- Form fields: Agency Name (required), Website (optional), Industry (optional dropdown: digital marketing, full-service, performance, creative, boutique, other)
- Submit calls saveAgencyInfo
- "Next" button

**Step 2: First Client Workspace**

- Explain: "Create your first client workspace. You can add more later."
- Form field: Client Name (required)
- Submit calls createFirstClient
- Store returned clientId in wizard state
- "Next" button

**Step 3: Choose Your Plan**

- Display AGENT_TIERS as cards (same layout as billing page tier cards)
- "Subscribe" button for each tier → calls createCheckoutSession, redirects to Stripe
- "Skip — Start Free" link to proceed without subscribing
- Note: After Stripe checkout, user returns to /billing?success=true, NOT back to wizard. The wizard stores progress, so on next visit to /onboarding it picks up at step 4.

**Step 4: Activate Skills**

- Show 6 skill cards with toggle switches
- Each toggle calls toggleClientSkill with the clientId from step 2
- Skills included in the chosen tier are pre-activated and marked as "Included in plan"
- Additional skills show add-on pricing
- "Complete Setup" button calls completeOnboarding, then redirects to /dashboard

**Wizard state management:**

- Use React useState for current step
- Persist wizard progress to localStorage (key: `fmai-onboarding-step`)
- On mount, read localStorage to resume from last completed step
- Store clientId from step 2 in localStorage for step 4

**UI/UX:**

- Progress bar at top showing 4 steps with labels
- Back button on steps 2-4
- Clean, centered layout with max-w-2xl
- Use framer-motion for step transitions (simple fade/slide)
- Consistent with dashboard dark theme
