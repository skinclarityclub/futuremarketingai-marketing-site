Create a new migration SQL file with the next sequential number. The migration must:

a) Extend `fma_organizations` with agency columns:

```sql
ALTER TABLE fma_organizations
  ADD COLUMN IF NOT EXISTS display_name TEXT,
  ADD COLUMN IF NOT EXISTS stripe_customer_id TEXT UNIQUE,
  ADD COLUMN IF NOT EXISTS stripe_subscription_id TEXT,
  ADD COLUMN IF NOT EXISTS onboarding_completed_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS agency_website TEXT,
  ADD COLUMN IF NOT EXISTS agency_industry TEXT;
```

b) Extend `fma_clients` with skill activation columns:

```sql
ALTER TABLE fma_clients
  ADD COLUMN IF NOT EXISTS active_skills TEXT[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS skill_config JSONB DEFAULT '{}';
```

c) Update subscription_tier CHECK constraint for AaaS tiers:

```sql
ALTER TABLE fma_organizations
  DROP CONSTRAINT IF EXISTS fma_organizations_subscription_tier_check;
ALTER TABLE fma_organizations
  ADD CONSTRAINT fma_organizations_subscription_tier_check
  CHECK (subscription_tier IN ('free', 'founding_member', 'starter', 'growth', 'agency'));
```

d) Add indexes for performance:

```sql
CREATE INDEX IF NOT EXISTS idx_fma_clients_active_skills ON fma_clients USING GIN (active_skills);
CREATE INDEX IF NOT EXISTS idx_fma_organizations_stripe_customer ON fma_organizations (stripe_customer_id) WHERE stripe_customer_id IS NOT NULL;
```

All columns use `IF NOT EXISTS` / defaults to avoid breaking existing SkinClarity data.
