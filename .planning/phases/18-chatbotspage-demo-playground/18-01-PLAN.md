**chatbotStore.ts — Migrate from global messageCount to per-persona messageCounts:**

1. Change `messageCount: number` to `messageCounts: Record<string, number>` in the interface and initial state (default `{}`).
2. Change `incrementMessageCount: () => void` to `incrementMessageCount: (personaId: string) => void`.
3. Implementation: `incrementMessageCount: (personaId) => set((state) => ({ messageCounts: { ...state.messageCounts, [personaId]: (state.messageCounts[personaId] || 0) + 1 } }))`.
4. Add `getMessageCount: (personaId: string) => number` as a derived getter. Implementation: a standalone selector function (not in store actions, since Zustand selectors are better for derived values). OR add it as a method that reads from state.
5. Keep `messageCount` as a computed getter for backward compatibility: `get messageCount() { return this.messageCounts[this.activePersonaId] || 0 }`. Since Zustand doesn't support getters in create(), instead add a standalone export: `export const getMessageCount = (personaId: string) => useChatbotStore.getState().messageCounts[personaId] || 0`.
6. Update `resetSession` to reset `messageCounts: {}` instead of `messageCount: 0`.
7. Update `partialize` to persist `messageCounts` instead of `messageCount`.

**usePersonaChat.ts — Read per-persona count:**

1. Replace `messageCount` destructuring with `messageCounts` from store.
2. Compute local: `const messageCount = messageCounts[personaId] || 0`.
3. In `onFinish`: call `incrementMessageCount(personaId)` (passing personaId).
4. Return `messageCount` and `isAtLimit` computed from per-persona count.
5. Keep DEMO_MESSAGE_LIMIT at 15.

These changes are backward-compatible. ChatWidget reads messageCount and isAtLimit from usePersonaChat return value — no ChatWidget changes needed.
