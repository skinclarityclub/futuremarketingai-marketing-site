**ProgressiveCTA.tsx — Message-count-based CTA banners:**

1. Props: `{ messageCount: number }`.
2. Use `useTranslation('chatbots')`.
3. Import CTAButton from `../common`.
4. Import `motion` from `framer-motion` for enter animation.

**Threshold logic:**

- `messageCount < 5`: return `null` (no CTA).
- `5 <= messageCount < 10`: Subtle banner with text link.
- `10 <= messageCount < 15`: Strong banner with Calendly CTAButton.
- `messageCount >= 15`: Gate banner — demo limit reached, Calendly CTA.

**Strong banner (10-14):**

**MultiPlatformShowcase.tsx — Animated architecture diagram with CSS-only animations:**

1. Use `useTranslation('chatbots')`.
2. Import Lucide icons: Brain, Globe, ShoppingBag, MessageCircle (for Claude AI, Website, Shopify, WhatsApp).
3. Import ScrollReveal from `../common/ScrollReveal`.

**Layout structure:**

```
Section heading: t('multi_platform.title') + t('multi_platform.subtitle')

Diagram (desktop): Flexbox horizontal
  [Brain node] ——— [Platform nodes column]

Diagram (mobile): Flexbox vertical
  [Brain node]
      |
  [Platform nodes stacked]
```

**Central brain node:**

- Circular div with `brain-pulse` CSS animation (defined inline via style or add to index.css).
- Brain icon (Lucide) + "Claude AI" label below.
- Animation: `@keyframes brainPulse { 0%,100% { box-shadow: 0 0 20px rgba(0,212,170,0.3) } 50% { box-shadow: 0 0 40px rgba(0,212,170,0.6) } }`.
- Use inline style for the animation to avoid adding to global CSS. Or use Tailwind `animate-` with a custom keyframe.

**Connection lines:**

- Use SVG with dashed lines (`stroke-dasharray`) and `stroke-dashoffset` animation for a "data flow" effect.
- SVG positioned between brain and platforms.
- On mobile, use vertical lines. On desktop, horizontal lines.
- Keep it simple: 3 lines from brain center to each platform node.
- Alternatively, use simple CSS borders/pseudo-elements instead of SVG for simplicity. A `div` with border-top between brain and each platform works well and is more maintainable.

**Decision:** Use CSS dividers (border + animated gradient pseudo-element) instead of SVG. Simpler, responsive-friendly.

**Platform nodes (3):**

- Define `PLATFORMS` array:
  ```typescript

  ] as const
  ```
- Each node: icon + name from `t('multi_platform.platforms.${id}.name')` + behavior from `t('multi_platform.platforms.${id}.behavior')`.
- Card styling: `card-gradient-border bg-white/[0.02] border border-border-primary rounded-card p-4`.
- Staggered animation on scroll via ScrollReveal or Framer Motion whileInView with delay.

**Connection animation (CSS-only per CONTEXT.md):**

- Each connection line is a div with `h-0.5 bg-gradient-to-r from-accent-system/50 to-accent-system` that animates width from 0 to full via CSS `@keyframes expandLine { from { width: 0 } to { width: 100% } }`.
