**1. Create `src/lib/chatbot/knowledge/flagship-kb.ts`:**
- Import ALL TopicDefinition arrays from the 5 existing KB files (concierge-kb, ecommerce-kb, leadgen-kb, support-kb, demo-guide-kb).
- Merge all topic definitions into a single `FLAGSHIP_TOPICS: TopicDefinition[]` array.
- No deduplication needed -- the topic router already filters by keyword relevance, so irrelevant topics won't bloat the prompt.
- Export `FLAGSHIP_TOPICS`.

**2. Create `src/lib/chatbot/tools/flagship-tools.ts`:**

- Import tools from all 5 tool files (concierge-tools, ecommerce-tools, leadgen-tools, support-tools, demo-guide-tools).
- Merge into a single `flagshipTools` record. Apply deduplication per the RESEARCH.md mapping:
  - `navigate_to_page`: Use concierge version but merge the route enum to include all pages + demo pages from demo-guide version. Keep the tool name the same but expand the Zod input schema to accept all routes.
  - `book_call`: Use concierge version as the single booking tool. Remove `schedule_demo` (leadgen) and `book_demo` (demo-guide) since they do the same thing.
  - `get_roi_info` (demo-guide) vs `get_roi_estimate` (leadgen): Keep BOTH -- they serve different purposes (simple overview vs detailed estimate).
  - All other tools: include as-is from their source persona (search_products, get_product_details, build_routine, qualify_lead, get_pricing_info, search_knowledge_base, create_ticket, check_status, escalate_to_human, get_services, get_case_study, explain_module).
- Result: ~16 unique tools in one `AnyToolRecord`.
- Export `flagshipTools`.
- Use the same `AnyToolRecord` type pattern from tool-executor.ts.

**3. Create `src/lib/chatbot/personas/flagship.ts`:**

- Write a CURATED system prompt (not concatenated). The prompt should:
  - Define the role: "FMai Flagship Concierge -- the most capable AI assistant on the FMai website"
  - List capabilities grouped by category (services, e-commerce demo, lead qualification, support, marketing machine modules, navigation, booking)
  - Include context-awareness instructions: check user's current page, suggest relevant tools, reference user journey
  - Communication style: professional, concise (2-4 sentences), bold for key terms, bullets for lists, no markdown headers
  - Decision rules: page-specific tool priorities, when to suggest booking, handling off-topic, complex question handling
  - Keep prompt under ~2000 tokens for efficient caching
- Set `defaultModel: 'haiku'`, `maxTokens: 600`, `temperature: 0.7`
- Use meaningful complexityKeywords: 'compare', 'ROI', 'pricing strategy', 'enterprise', 'custom solution', 'integration', 'technical details'
- Set tools record with all tool names mapped to `true` for validation
- Call `registerPersona(flagshipPersona)` at module scope
- Export `flagshipPersona` and `FLAGSHIP_STARTERS` (page-context-aware conversation starters in EN/NL/ES format matching existing persona starter pattern)
