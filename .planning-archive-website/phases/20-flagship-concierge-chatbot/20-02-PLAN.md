- For mobile (below lg breakpoint): render as a fixed full-screen modal overlay instead of an inline side panel. Use a Tailwind `lg:` responsive approach:
  - On mobile: position fixed, inset-0, z-50, with bg-bg-surface and a semi-transparent backdrop.
  - On desktop: the existing inline flex child approach.
  - Use a wrapper that switches layout based on screen size. The simplest approach: always render inline on desktop (as flex child of chat container), and on mobile render as a portal/fixed overlay. Use a CSS media query approach via Tailwind classes rather than JS detection.
