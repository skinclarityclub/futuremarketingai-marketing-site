**Create `src/components/chatbot/ChatMessages.tsx`:**

```ts
interface ChatMessagesProps {
  messages: UIMessage[] // from 'ai' package

}
```

Import `UIMessage` type from `ai` package. Import `ToolResultRenderer` from `./tool-results` (created in Plan 03 — for now, create a placeholder inline that renders `<div>Tool: {part.toolName}</div>`).

**Message rendering — CRITICAL: use `message.parts` not `message.content`:**

```tsx
{
  messages.map((message) => (
    <div
      key={message.id}
      className={message.role === 'user' ? 'flex justify-end' : 'flex justify-start'}
    >
      <div className={message.role === 'user' ? userBubbleClass : assistantBubbleClass}>
        {message.parts.map((part, i) => {
          if (part.type === 'text') {
            return <MarkdownContent key={i} text={part.text} />
          }
          if ('toolName' in part) {
            return <ToolResultPlaceholder key={i} part={part} />
          }
          return null
        })}
      </div>
    </div>
  ))
}
```

**MarkdownContent** — inline component using `react-markdown` (already installed):

- Renders message text with basic markdown (bold, lists, links)
- Links open in new tab with `target="_blank" rel="noopener noreferrer"`
- Style links with `text-accent-system hover:underline`
- Prose styling: `text-sm leading-relaxed`

**ToolResultPlaceholder** — temporary inline component until Plan 03:

- When `part.state === 'input-streaming' || part.state === 'input-available'`: show a loading pill with `animate-pulse` and tool name
- When `part.state === 'output-available'`: show `<pre>{JSON.stringify(part.output, null, 2)}</pre>` as placeholder
- When `part.state === 'output-error'`: show error text in `text-red-400`

**Typing indicator** — shown when `status === 'submitted'` (waiting for first token):

- Three animated dots in a glass bubble: `bg-bg-elevated/80 backdrop-blur-md border border-border-primary`
- Dots use CSS animation (staggered bounce): define inline keyframes or use tailwind animate-bounce with staggered delays

**Auto-scroll** — `useRef` on scroll container + `useEffect` on messages length:

- `messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })`
- Only auto-scroll if user hasn't manually scrolled up (track with `onScroll` — if scrollTop is near bottom, auto-scroll; if user scrolled up, don't)

**Styling:**

- Container: `flex-1 overflow-y-auto px-4 py-4 space-y-4`
- User bubble: `max-w-[85%] px-4 py-3 rounded-2xl rounded-tr-md bg-gradient-to-br from-accent-human/90 to-accent-human/70 text-white text-sm`
- Assistant bubble: `max-w-[85%] px-4 py-3 rounded-2xl rounded-tl-md bg-bg-elevated/80 backdrop-blur-md border border-border-primary text-text-primary text-sm`
- Each message has CSS `animate-fadeIn`
