**Modify `src/stores/chatbotStore.ts`:**

Add demo mode state fields to the ChatbotState interface (after the tracking state):

- `demoMode: boolean` (default: false)
- `demoScenarioId: string | null` (default: null)
- `demoStepIndex: number` (default: 0)
- `demoStatus: 'idle' | 'choosing' | 'running' | 'checkpoint' | 'completed'` (default: 'idle')
- `demoStartedAt: number | null` (default: null)

Add demo actions:

- `startDemo()` — sets demoMode: true, demoStatus: 'choosing', opens chat panel (isOpen: true, isMinimized: false)
- `selectScenario(id)` — sets demoScenarioId, resets demoStepIndex to 0, sets demoStatus: 'running', records demoStartedAt: Date.now()
- `advanceStep()` — increments demoStepIndex, sets demoStatus: 'running'
- `setDemoStatus(status)` — updates demoStatus
- `endDemo()` — resets all demo fields to defaults

**IMPORTANT:** Demo state is ephemeral — do NOT include it in the `persist` partialize config. These fields should reset on page reload.

Use the exact implementation from `docs/plans/2026-03-16-guided-demo-implementation.md` Task 2.
