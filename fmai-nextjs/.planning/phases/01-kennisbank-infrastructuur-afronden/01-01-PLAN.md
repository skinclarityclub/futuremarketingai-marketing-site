**DefinedTermSetJsonLd.tsx** — a Server Component emitter following the WebPageJsonLd pattern. Props: `{ locale: string; terms: { id: string; name: string; definition: string }[] }`. Build:
```typescript
const data = {
  '@context': 'https://schema.org',
  '@type': 'DefinedTermSet',
  '@id': `${SITE_URL}/${locale}/resources#glossary`,
  name: 'GEO & AI marketing glossary',
  hasDefinedTerm: terms.map((term) => ({
    '@type': 'DefinedTerm',
    '@id': `${SITE_URL}/${locale}/resources#${term.id}`,
    name: term.name,
    description: term.definition,
    inDefinedTermSet: `${SITE_URL}/${locale}/resources#glossary`,
  })),
}
return <JsonLd data={data} />
```
Import `SITE_URL` from `@/lib/seo-config` and `JsonLd` from `@/components/seo/JsonLd`. Type the data object explicitly (Record<string, unknown> is acceptable for the JsonLd prop) — no `any`.

**Glossary.tsx** — a pure presentational component (no 'use client'; receives resolved copy as a prop so the page owns translation). Props: `{ terms: { id: string; name: string; definition: string }[]; heading: string; intro: string }`. Render:
